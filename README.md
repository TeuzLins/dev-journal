# dev-journal

> Registro diário da minha evolução como desenvolvedor, reunindo estudos, práticas, ideias, melhorias de código e aprendizados ao longo da jornada.

## Sobre o projeto

Este repositório funciona como um diário técnico automatizado, criado para acompanhar minha constância nos estudos e no desenvolvimento de projetos.

A ideia é simples: manter um histórico organizado da minha evolução como desenvolvedor, registrando diariamente pequenos avanços, aprendizados e atividades relacionadas à programação.

## Objetivo

* Registrar minha evolução diária como desenvolvedor
* Criar consistência nos estudos e práticas de programação
* Manter um histórico público de aprendizado
* Organizar ideias, melhorias e anotações técnicas
* Demonstrar disciplina e progresso contínuo no GitHub

## Tecnologias e ferramentas

Este repositório utiliza:

* GitHub Actions
* Git
* Markdown
* Automação com workflow agendado

## Como funciona

O repositório possui um workflow automatizado com GitHub Actions que atualiza o arquivo `dev-log.md` diariamente.

Esse processo adiciona uma nova entrada com a data do dia, mantendo um registro contínuo da minha jornada de estudos e desenvolvimento.

## Estrutura

```txt
dev-journal/
├── .github/
│   └── workflows/
│       └── daily-dev-log.yml
├── dev-log.md
└── README.md
```

## Dev Log

O arquivo principal de registro é:

```txt
dev-log.md
```

Nele ficam armazenadas as atualizações diárias geradas automaticamente.

## Por que isso existe?

Porque evolução em tecnologia não acontece do dia para a noite.

Este repositório representa constância, disciplina e acompanhamento de progresso. Pequenos registros diários ajudam a visualizar a caminhada, manter o foco e reforçar o hábito de continuar aprendendo.
